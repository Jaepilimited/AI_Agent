# LLM 데이터 처리 메커니즘 상세 — 보안팀 회신용

## 동작 원리

```
┌─ 사내 서버 (172.16.1.250) ─────────────────────────────┐
│                                                         │
│  사용자 질문 → FastAPI → 프롬프트 조립 → HTTPS 전송 ──────┼──→ 외부 LLM API 서버
│                                                         │     (Anthropic / Google)
│                                         ←── 응답 수신 ──┼──← 응답 생성 후 반환
│                                                         │
│  응답 텍스트 → 사용자 화면 표시                            │
│             → MariaDB 저장 (대화 이력)                    │
└─────────────────────────────────────────────────────────┘
```

## LLM API의 데이터 처리 방식

LLM은 **이미 학습이 완료된 고정 모델**입니다. API 호출 시:

```
1. 요청 수신: 프롬프트(질문+맥락) 수신
2. 추론 실행: 기존 학습된 가중치로 토큰 생성 (모델 변경 없음)
3. 응답 반환: 생성된 텍스트 전송
4. 메모리 해제: 요청 관련 데이터 GPU 메모리에서 소멸
```

**핵심: 추론(inference)만 수행하며, 우리 데이터로 모델을 재학습(fine-tuning)하지 않습니다.**

일반적으로 알려진 "AI가 학습한다"는 개념과 다릅니다:
- ❌ 우리 질문으로 모델이 똑똑해지는 것 아님
- ❌ 우리 매출 데이터를 기억하는 것 아님
- ✅ 매번 백지 상태에서 프롬프트만 보고 답변 생성

## API vs 웹 서비스 차이

| | ChatGPT 웹 (chat.openai.com) | 우리 시스템 (API 호출) |
|---|---|---|
| 대화 저장 | OpenAI 서버에 저장 | **OpenAI에 저장 안 됨** |
| 모델 학습 | 학습에 활용 가능 (opt-out 가능) | **학습에 절대 사용 안 함** |
| 데이터 보존 | 계정에 영구 보존 | **API 로그만 30일 보존 후 삭제** |
| 접근 주체 | OpenAI 직원 열람 가능 | **남용 감사 목적 외 열람 불가** |

## 각 LLM 제공사의 API 데이터 정책

**Anthropic (Claude) — 대화용 LLM:**
- API 입출력 데이터: **모델 학습에 사용하지 않음** (API Terms of Service 명시)
- 안전 로그: 요청 데이터를 **최대 30일간 보관** (정책 위반 감사 목적)
- 30일 후: **자동 삭제**
- 직원 열람: 안전 사고 조사 시에만 제한적 열람
- 근거: Anthropic API Usage Policy — "We do not train on your API inputs and outputs"

**Google (Gemini) — SQL 생성/차트용 LLM:**
- 유료 API(Vertex AI / AI Studio 유료 플랜): **학습에 사용하지 않음**
- 데이터 보존: **요청 처리 후 즉시 삭제** (로그 미보존)
- 근거: Google Cloud Data Processing Terms

## 외부 전송되는 데이터 구체적 예시

**예시 1: 매출 질문**
```
사용자: "이번 달 인도네시아 매출 알려줘"

Gemini에 전송:
  - SQL 생성 프롬프트 (테이블 스키마 + 규칙 + 질문)
  - ※ 실제 매출 수치는 이 단계에서 전송 안 됨

BigQuery에서 조회:
  - 결과: [{Country: "인도네시아", revenue: 5000000000}]

Gemini에 전송 (답변 포맷):
  - "사용자 질문: 이번 달 인도네시아 매출" + 조회 결과 JSON
  - ※ 이 시점에서 매출 수치가 Gemini에 전송됨

Gemini 응답:
  - "인도네시아 이번 달 매출은 약 50억원입니다..."
  - → 사용자에게 표시 + MariaDB 저장 (답변 텍스트)
  - → Gemini 서버에서 메모리 해제 (저장/학습 없음)
```

**예시 2: 일반 대화**
```
사용자: "우리 회사 뭐하는 회사야"

Claude에 전송:
  - 시스템 프롬프트 (회사 정보 포함) + 질문 텍스트
  - ※ BigQuery 데이터 전송 없음

Claude 응답:
  - "크레이버코퍼레이션은 SKIN1004 브랜드를..."
  - → 사용자에게 표시 + MariaDB 저장
  - → Anthropic 서버: 30일 로그 후 삭제
```

## 정보 저장 가능성 요약

| 위치 | 저장 여부 | 내용 | 보존 기간 |
|------|----------|------|----------|
| **사내 MariaDB** | ✅ 저장 | 질문/답변 텍스트, SQL 캐시 | 설정 가능 (현재 무기한) |
| **사내 서버 파일** | ✅ 저장 | GWS OAuth 토큰 | 토큰 만료 시까지 |
| **Anthropic 서버** | ⚠️ 임시 보관 | API 요청/응답 로그 | **30일 후 자동 삭제** |
| **Google 서버** | ❌ 미저장 | 유료 API는 즉시 삭제 | 즉시 |
| **LLM 모델 가중치** | ❌ 반영 안 됨 | 학습에 사용하지 않음 | - |

## 잔존 리스크 및 권장 조치

1. **Anthropic 30일 로그**: API 요청에 BQ 조회 결과(매출 수치 등)가 포함되므로, 민감도 높은 데이터의 경우 **전송 전 마스킹** 또는 **요약 전송** 검토
2. **네트워크 구간**: 사내 서버 → 외부 API 간 HTTPS 암호화 (TLS 1.3) 적용 중
3. **사내 MariaDB**: 대화 이력에 매출 수치가 텍스트로 남으므로 **접근 권한 및 보존 정책** 수립 권장
4. **향후 대안**: 민감 데이터 처리 시 **온프레미스 LLM** (Llama 등) 도입으로 외부 전송 완전 차단 가능
