## 5. 버그 수정 상세 (v6.1 / v4.1)

### 5.1 Notion Agent v6.1 (4건 수정)

**수정 1: Google Sheet 읽기 타임아웃 (NT-03, R2-NT-01)**
- 원인: `asyncio.to_thread(read_google_sheet)` 호출에 timeout 없음 → 대형 시트에서 300s+ 소요
- 수정: `asyncio.wait_for(..., timeout=30.0)` 래핑 + `max_rows=50` 제한
- 적용 위치: `_read_page_blocks`, `_read_database_entries`, `_read_page_fallback` 총 3곳

**수정 2: httpx 클라이언트 재생성 (NT-04)**
- 원인: `_close_client()`에서 이미 닫힌 client의 `self._client = None` 미설정 + `_request_with_retry`에서 retry 시 stale local 변수 사용
- 수정: (1) `_close_client`가 항상 `self._client = None` 설정, (2) retry 루프 매 시도마다 `client = await self._get_client()` 재호출

**수정 3: 검색어 구두점 매칭 (R2-NT-09)**
- 원인: 사용자 질문 "쇼피, 틱톡샵, 아마존"에서 쉼표가 검색어에 포함 → "틱톡샵,"가 페이지 제목 "틱톡샵"과 불일치
- 수정: `re.sub(r'[^\w가-힣a-zA-Z0-9]', '', word)` 로 검색어에서 구두점 제거

### 5.2 GWS Agent v4.1 (1건 수정)

**수정 4: ReAct 에이전트 타임아웃 (GWS-05)**
- 원인: `create_react_agent`의 `ainvoke()`에 timeout 없음 → 복잡한 검색에서 300s+ 소요
- 수정: `asyncio.wait_for(agent.ainvoke(...), timeout=120.0)` + `TimeoutError` 시 사용자 안내 메시지 반환 + "도구 호출 최소화" 시스템 프롬프트 추가

### 5.3 수정된 파일

|------|-----------|
| `app/agents/notion_agent.py` | v6.0 → v6.1: Sheet timeout, client 재생성, 구두점 제거 |
